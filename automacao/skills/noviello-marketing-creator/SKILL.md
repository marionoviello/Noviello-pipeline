**REGRA CRÍTICA PARA DOCUMENTOS FORMAIS (orçamentos, propostas, pareceres, ofícios, contratos de honorários):**

Construa SEMPRE sobre `assets/papelaria/papel-de-carta-padrao.docx`. Nunca crie layouts alternativos. Workflow técnico:

```bash
# 1. Desempacotar preservando header/footer/mídias
python /mnt/skills/public/docx/scripts/office/unpack.py \
  assets/papelaria/papel-de-carta-padrao.docx unpacked/

# 2. Editar word/document.xml injetando conteúdo no body (header/footer intactos)

# 3. Reempacotar validando
python /mnt/skills/public/docx/scripts/office/pack.py \
  unpacked/ output.docx --original assets/papelaria/papel-de-carta-padrao.docx
```

**Ordem XML obrigatória em `<w:pPr>`:** `pStyle` → `keepNext` → `pageBreakBefore` → `pBdr` → `shd` → `spacing` → `ind` → `jc` → `rPr`

**Ordem XML obrigatória em `<w:tcPr>`:** `tcW` → `gridSpan` → `tcBorders` → `shd` → `tcMar` → `vAlign`

**Antes de gerar PDF do .docx** (via LibreOffice/soffice), instalar as fontes:
```bash
mkdir -p ~/.fonts && cp assets/fonts/*.ttf ~/.fonts/ && fc-cache -f ~/.fonts/
```
